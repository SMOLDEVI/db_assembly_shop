# Сборочный цех — Система учета

Веб-приложение на базе **Flask** и **SQLite** для автоматизации процессов сборочного производства, адресного складского учета комплектующих и готовой продукции, а также ведения планов выпуска с расчетом дефицита деталей.

Интерфейс оформлен в спокойной цветовой схеме (Slate Blue / Sage Green) для минимизации нагрузки на зрение при продолжительной работе с таблицами.

---

## 🛠️ Технологический стек
- **Backend:** Python 3, Flask, SQLite3
- **Frontend:** HTML5 (Jinja2), CSS3, Bootstrap 5, Bootstrap Icons, Chart.js
- **Документирование:** python-docx, pypdf (генератор отчетов по ГОСТ)
- **Контейнеризация:** Docker, Docker Compose

---

## 📂 Структура проекта
*   `app.py` — основной сервер Flask, описывающий маршруты и логику списания/планирования.
*   `init_db.py` — скрипт для инициализации базы данных `database.db` демонстрационной спецификацией и остатками.
*   `make_coursework.py` — генератор пояснительной записки курсового проекта в формате Word (`Курсовая_работа_Сборочный_цех.docx`) по шаблону РГАТУ.
*   `templates/` — папка HTML-шаблонов веб-приложения.
*   `Dockerfile` и `docker-compose.yml` — конфигурация для контейнеризации приложения.
*   `requirements.txt` — список необходимых Python-библиотек.

---

## 🚀 Вариант 1. Запуск через Docker (Рекомендуемый)

Если у вас установлен Docker, проект можно запустить одной командой без необходимости устанавливать Python и библиотеки локально.

1.  Убедитесь, что Docker Desktop запущен.
2.  Выполните команду сборки и запуска в терминале (в корне папки проекта):
    ```bash
    docker-compose up --build
    ```
3.  Откройте браузер и перейдите по адресу:
    ```
    http://localhost:5000
    ```
4.  Для остановки контейнеров нажмите `Ctrl + C` в терминале или выполните:
    ```bash
    docker-compose down
    ```

## 💻 Вариант 2. Локальный запуск на компьютере

### Шаг 1. Создание и активация виртуального окружения
Откройте терминал в папке проекта и создайте изолированное окружение:

**На Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

**На macOS / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### Шаг 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

### Шаг 3. Инициализация базы данных
Запустите скрипт, который создаст файл `database.db` и заполнит его начальной номенклатурой (детали, спецификации ПК, ячейки хранения и текущий план):
```bash
python init_db.py
```

### Шаг 4. Запуск веб-приложения
```bash
python app.py
```
Перейдите по адресу `http://127.0.0.1:5000` в браузере.

## 📝 Дополнительно: Генерация пояснительной записки Word

Вы можете сгенерировать Word-документ пояснительной записки курсовой работы по ГОСТу с описанием этой базы данных:
```bash
python make_coursework.py
```
В корневой папке будет создан или перезаписан файл `Курсовая_работа_Сборочный_цех.docx`.
