2.  Постановка  задачи   
Торговая  фирма   Торговая  фирма  занимается  оптовой  торговлей.  Фирма  имеет  сеть   поставщиков  товаров.  Поставщики  находятся  в  разных  городах.  Каждый  
поставщик
 
специализируется
 
на
 
поставке
 
одного
 
товара,
 
склады
 
фирм
 
позволяют
 
разместить
 
ограниченное
 
количество
 
каждого
 
из
 
товаров.
 Товар  на  фирму  может  поступать  по  ее  заявкам  или  по  предложению   поставщика  без  предварительного  заказа.  Отпуск  товаров  производится  по  
заявкам.
 
 Система  должна  обеспечивать  выполнение  следующих  операций:  -  прием  предложения  поставщика  о  возможности  поставить  товар  в  
виде
 
накладной;
 -  прием  товара  на  склад  или  отказ  в  приеме;  -  формирование  заказа  поставщику  на  поставку  товара;  -  отпуск  товара  или  отказ;  -  получение  справок  о  сделанных  заказах  поставщикам,  
предложениях
 
поставщиков,
 
о
 
заявках
 
на
 
отпуск
 
товаров;
 -  получение  других  справок.

3.  Описание  предметной  области    
Общее  описание  предметной  области    При  процессе  подачи  заявки  на  товар  клиент  обращается  в  торговую  
фирму,
 
где
 
заполняет
 
специальную
 
форму,
 
указывая
 
необходимые
 
товары
 
и
 
их
 
количество.
 
После
 
заполнения

клиент
 
передает
 
ее
 
сотруднику
 
фирмы,
 
который
 
затем
 
принимает
 
заявку,
 
проверяя
 
её
 
корректность
 
и
 
наличие
 
необходимых
 
товаров
 
на
 
складе.
 
Если
 
данные
 
в
 
заявке
 
корректны
 
и
 
товары
 
имеются,
 
сотрудник
 
принимает
 
заявку
 
для
 
дальнейшей
 
обработки.
  Предметная  область  подразумевает  выполнение  следующих  
функций:
 ●  Добавление  нового  товара;  ●  Удаление  товара  из  списка;  ●  Добавление  поставщика;  ●  Удаление  поставщик;  ●  Добавление  заказа;  ●  Удаление  заказа  ●  Добавление  склада;  ●  Удаление  склада;  ●  Просмотр  подробностей  о  всех  данных  из  столбцов  ●  Отмену  или  принятие  заказа  от  клиента;  ●  Просмотр  подробностей  о  товаре.  ●  Просмотр  подробностей  о  складе.  ●  Просмотр  подробностей  о  поставщике.  ●  Просмотр  подробностей  о  заказе.  ●  Отмену  или  принятие  заявки  от  клиента.  ●  Диаграмма  количества  имеющихся  товаров  в  процентном  
соотношении.
 ●  Отчет,  формируемый  о  товаре  в  формате  .csv

4.  Инфологическая  модель   
В  ходе  изучения  одной  из  заявок  торговой  фирмы  мы  пришли  к  
выводу,
 
что
 
самым
 
оптимальным
 
выбором
 
для
 
создания
 
базы
 
данных
 
будет
 
тот,
 
что
 
описан
 
далее.
 
Он
 
будет
 
самым
 
оптимальным
 
и
 
универсальным.
 
 Сущности  и  их  ключевые  атрибуты:  
 ●  Товар  ( кодТовара )  ●  Склад  ( кодСклад )  ●  Заявка  ( кодЗаявка )  ●  Поставщик  товара  ( кодПоставщик )  ●  Логин  ( Логин )  Бизнес  правила   Заявка-Товар  ●  Заявка  может  включать  в  себя  несколько  товаров.  ●  Товар  может  быть  включен  в  несколько  заявок.  ●  Товар  может  быть  не  включен  ни  в  одну  заявку.  ●  Заявка  должна  включать  хотя  бы  один  товар.  Склад-Товар  ●  Товар  может  быть  размещен  на  нескольких  складах.  ●  На  складе  может  быть  размещено  несколько  товаров.  ●  Товар  не  обязательно  размещен  на  складе.  ●  На  складе  должен  быть  размещен  хотя  бы  один  товар.  Товар-Поставщик  товара   ●  Товар  может  быть  поставлен  несколькими  поставщиками.  ●  Поставщик  может  поставлять  только  один  товар.  ●  Товар  должен  поставляться  каким-либо  поставщиком.  ●  Поставщик  должен  поставлять  какой-либо  товар.  Логин-Клиент  ●  Клиент  может  иметь  только  один  логин.  ●  Логин  может  принадлежать  только  одному  клиенту.  ●  У  клиента  обязательно  должен  быть  логин.  ●  Каждому  логину  должен  соответствовать  клиент.

6.  Реализация  базы  данных .   Работа  с  базой  данных  осуществляется  с  помощью  встроенного  движка  
PostgreSQL.
 
В

 (API)  библиотеки  SQLite.  Данные  обработчики  всегда  вызывают  функции,  которые  формируют  SQL– запросы  к  СУБД  и  возвращают  ответы  в  виде  
списка

данных.
  СУБД  не  использует  ни  триггеры,  ни  хранимые  процедуры.  Вес  
запросы
